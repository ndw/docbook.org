holder — The name of the individual or organization that holds a copyright

Synopsis

Mixed Content Model

holder ::=
(#PCDATA|link|olink|ulink|emphasis|trademark|replaceable|remark|
 subscript|superscript|inlinegraphic|inlinemediaobject|indexterm)*

Attributes

Common attributes

DocBook NG “Bourbon” Content Model

holder ::=
  • Zero or more of:
    • text
    • anchor
    • indexterm (indexterm.endofrange)
    • indexterm (indexterm.singular)
    • indexterm (indexterm.startofrange)
    • inlinemediaobject
    • phrase (text.phrase)
    • remark
    • replaceable

DocBook NG “Bourbon” Attributes

Common attributes and common linking attributes.

Description

Holder in Copyright identifies an individual or organization that asserts a copyright on the document.

Processing expectations

The formatting of Holder depends on the formatting of its parent Copyright. In the case of a Copyright with multiple holders, additional punctuation may need to be generated when Holder is processed.

Parents

These elements contain holder: copyright.

Children

The following elements occur in holder: emphasis, indexterm, inlinegraphic, inlinemediaobject, link, olink, remark, replaceable, subscript, superscript, trademark, ulink.

Examples

For examples, see bibliography, biblioset, bookinfo, copyright.
